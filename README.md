# Telegram-бот напоминалка для аниматоров

Бот **каждый день в 12:00** (по таймзоне из `.env`) автоматически отправляет в указанный чат сообщение со списком анимационных программ **на следующий день**, беря данные из Google Sheets.

## Что нужно подготовить

### 1) Telegram

1. Создайте бота у `@BotFather`, получите токен.
2. Добавьте бота в нужный чат/группу/канал.
3. Узнайте `CHAT_ID`:
   - **Группа/супергруппа**: самый простой способ — временно добавить `@userinfobot` и посмотреть `id`, либо открыть обновления через любой getUpdates-инструмент.
   - **Канал**: обычно `CHAT_ID` выглядит как `-100...`. Убедитесь, что бот добавлен в канал и имеет право писать сообщения.

### 2) Google Sheets + credentials.json

Бот использует **Service Account**.

1. В Google Cloud Console создайте проект (или используйте существующий).
2. Включите API:
   - Google Sheets API
   - Google Drive API (нужно, чтобы читать таблицу по имени через Drive)
3. Создайте **Service Account**, скачайте ключ в формате JSON.
4. Положите файл ключа в корень проекта и назовите **`credentials.json`** (или укажите путь в `GOOGLE_CREDENTIALS_PATH`).
5. Откройте вашу Google-таблицу и **поделитесь** ею с email сервисного аккаунта (внутри `credentials.json`, поле `client_email`) с правом просмотра (Viewer) или выше.

### 3) Структура таблицы

Ожидаются столбцы (названия можно с небольшими вариациями пробелов/регистра):

- `Дата заказа`
- `Дата/время`
- `мер-ия`
- `Персонаж`
- `ФИО аниматора`
- `комментарий`

Фильтрация идёт по **дате из `Дата/время`** (если она парсится) иначе по `Дата заказа`.

Поддерживаемые форматы:
- Дата: `DD.MM.YYYY`, `YYYY-MM-DD`, `DD/MM/YYYY`
- Дата/время: `DD.MM.YYYY HH:MM`, `YYYY-MM-DD HH:MM`, `DD/MM/YYYY HH:MM` (секунды опционально)

## Установка и запуск (Windows)

1. Установите Python 3.10+.
2. В папке проекта создайте виртуальное окружение и установите зависимости:

```bash
python -m venv .venv
.\.venv\Scripts\pip install -r requirements.txt
```

> Если PowerShell ругается на `activate` (Execution Policy), это нормально: **активация не обязательна**.
> Можно либо всегда вызывать `.\.venv\Scripts\python`/`pip`, либо один раз разрешить скрипты:
>
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
>
> После этого активация будет работать:
>
> ```powershell
> .\.venv\Scripts\activate
> ```

3. Создайте файл `.env` рядом с `bot.py` (можно скопировать из `.env.example`) и заполните:

- `TELEGRAM_TOKEN`
- `CHAT_ID`
- `GOOGLE_SHEET_NAME`
- (опционально) `TIMEZONE` (по умолчанию `Europe/Moscow`)

Важно: пишите значения **без кавычек** и без лишних пробелов, например:
`TELEGRAM_TOKEN=123456:ABCDEF...`, а не `TELEGRAM_TOKEN=" 123456:ABCDEF..."`.

`GOOGLE_SHEET_NAME` можно указать как:
- название документа в Google Drive (точь-в-точь)
- **или ссылку на таблицу**
- **или ID таблицы** (часть после `/d/` в ссылке)

4. Убедитесь, что рядом лежит `credentials.json`.
5. Запуск:

```bash
.\.venv\Scripts\python bot.py
```

После запуска бот будет работать автономно и отправлять сообщение **каждый день в 12:00**.

## Частые ошибки (и почему “не запускается”)

### 1) Вставка ```bash в PowerShell

Если вы копируете команды вместе с строкой ```bash, PowerShell пытается выполнить слово `bash`, и вы увидите ошибку вида **`ash : Имя "ash" не распознано...`**.  
Решение: вставляйте **только сами команды**, без обрамления тройными кавычками.

### 2) `requirements.tx` вместо `requirements.txt`

Это обычная опечатка — файл называется **`requirements.txt`**.

### 3) `Telegram 409 Conflict: terminated by other getUpdates request`

Это означает, что **запущены 2 (или больше) копии бота с одним и тем же токеном** (или другой скрипт/служба, который делает `getUpdates`).
Telegram разрешает long polling (`getUpdates`) **только одному процессу**.

Что сделать:
- Остановите предыдущий запуск (обычно `Ctrl+C` в другой консоли, где вы ранее запускали `python bot.py`)
- Закройте лишние окна/терминалы, где бот мог остаться работать
- Если после аварийного завершения остался файл `.bot.lock` в папке проекта — удалите его

### 4) `Unable to copy ... venvlauncher.exe ... to .venv\\Scripts\\python.exe`

Обычно это значит, что папка `.venv` уже существует/частично создана или файл заблокирован (антивирус, запущенный python-процесс).

Быстрое решение:
- Закройте все запущенные `python` (где мог работать бот)
- Удалите папку `.venv`
- Создайте окружение заново: `python -m venv .venv`

## Примечания

- Если таблица недоступна/не читается, бот отправит в чат сообщение об ошибке.
- Если Telegram не принимает сообщение, ошибка будет записана в логи.
- По умолчанию используется первый лист (`sheet1`) в документе.
